
* **Тема:** Warehouse (Управління складським обліком)
* **Мета:** Створення двох незалежних сервісів на базі сучасних PHP-фреймворків для демонстрації міжплатформної розробки, побудови однакової логіки REST API, маршрутизації та генерації автоматичної документації для клієнтських додатків (Postman).

---

## 📌 Опис предметної області та сутності
Для автоматизації складу обрано ключову сутність — **Product** (Товар на складі). Кожен об'єкт системи містить унікальний набір характеристик, що дозволяє відстежувати залишки та вартість активів компанії:

* `id` (integer) — Унікальний внутрішній ідентифікатор запису в системі.
* `title` (string) — Зрозуміла назва товару.
* `sku` (string) — Артикул або унікальний код товарної одиниці.
* `quantity` (integer) — Поточна фізична кількість одиниць товару на складі.
* `price` (float) — Ціна за одну одиницю товару в базовій валюті.

## 🚀 Архітектура та Ендпоінти (API Маршрути)

Проекти налаштовані на роботу на різних локальних портах з метою уникнення конфліктів при одночасному запуску. Загалом реалізовано **8 ендпоінтів** (по 4 на кожен фреймворк):

### 1. Symfony Сервіс (Локальний порт `:8000`)
Базовий URL: `http://127.0.0.1:8000/api/warehouse`

* **`GET /products`**
  - *Призначення:* Повертає повний список наявних на складі товарів.
  - *Код відповіді:* `200 OK`
* **`POST /products`**
  - *Призначення:* Реєстрація (створення) нового товару на складі.
  - *Код відповіді:* `201 Created`
* **`PATCH /products/{id}`**
  - *Призначення:* Часткове оновлення інформації про товар (наприклад, коригування кількості або ціни).
  - *Код відповіді:* `200 OK`
* **`DELETE /products/{id}`**
  - *Призначення:* Списання/видалення товару з бази даних складу за ідентифікатором.
  - *Код відповіді:* `200 OK`

### 2. Laravel Сервіс (Локальний порт `:8001`)
Базовий URL: `http://127.0.0.1:8001/api`

* **`GET /products`**
  - *Призначення:* Отримання масиву всіх зареєстрованих товарів.
  - *Код відповіді:* `200 OK`
* **`POST /products`**
  - *Призначення:* Створення нової картки товару.
  - *Код відповіді:* `201 Created`
* **`PATCH /products/{id}`**
  - *Призначення:* Зміна окремих полів існуючого товару.
  - *Код відповіді:* `200 OK`
* **`DELETE /products/{id}`**
  - *Призначення:* Видалення товару з облікової системи.
  - *Код відповіді:* `200 OK`
